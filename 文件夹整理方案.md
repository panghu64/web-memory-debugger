## 📝 具体操作步骤

### 步骤1: 创建顶层分类文件夹（中文命名）
```powershell
New-Item -ItemType Directory -Force -Path "项目源码"
New-Item -ItemType Directory -Force -Path "研究分析"
New-Item -ItemType Directory -Force -Path "文档资料"
New-Item -ItemType Directory -Force -Path "工具脚本"
New-Item -ItemType Directory -Force -Path "归档文件"
New-Item -ItemType Directory -Force -Path ".ide"
New-Item -ItemType Directory -Force -Path "构建配置"
New-Item -ItemType Directory -Force -Path "临时文件"
```

### 步骤2: 移动项目源码
```powershell
Move-Item -Force "app" -Destination "项目源码/"
Move-Item -Force "myapplication" -Destination "项目源码/"
```

### 步骤3: 移动研究分析数据
```powershell
Move-Item -Force "doce" -Destination "研究分析/"
Move-Item -Force "ida_dumps" -Destination "研究分析/"
Move-Item -Force "data" -Destination "研究分析/" -ErrorAction SilentlyContinue
Move-Item -Force "tools" -Destination "研究分析/" -ErrorAction SilentlyContinue
```

### 步骤4: 重命名已整理的文件夹
```powershell
Rename-Item -Force "docs" -NewName "文档资料"
Rename-Item -Force "scripts" -NewName "工具脚本"
Rename-Item -Force "archive" -NewName "归档文件"
```

### 步骤5: 移动IDE配置（可选，隐藏复杂度）
```powershell
# 方案A：移动到.ide文件夹
Move-Item -Force ".config" -Destination ".ide/" -ErrorAction SilentlyContinue
Move-Item -Force ".cursor" -Destination ".ide/" -ErrorAction SilentlyContinue
# 注意：.gradle, .idea, .vs, .vscode 最好保留原位置，IDE需要

# 方案B：保持原样（推荐）
# 这些是IDE自动生成的，移动可能导致IDE配置丢失
```

### 步骤6: 移动构建配置和临时文件
```powershell
Move-Item -Force "gradle" -Destination "构建配置/"
Move-Item -Force "temp_check" -Destination "临时文件/" -ErrorAction SilentlyContinue
```

## ⚠️ 重要提示

### 不建议移动的文件夹
1. **.gradle** - Gradle缓存，移动会影响构建
2. **.idea** - IDEA配置，移动会丢失项目设置
3. **.vs** - Visual Studio配置
4. **.vscode** - VSCode配置
5. **.cursor** - Cursor配置
6. **gradle** - Gradle Wrapper，构建系统需要在根目录

### 建议的保守方案
只移动数据和文档类文件夹，保留IDE和构建配置在原位置：

**移动分类**：
- ✅ app, myapplication → 项目源码/
- ✅ doce, ida_dumps → 研究分析/
- ✅ docs, scripts, archive → 重命名为中文

**保留原位置**：
- 保留 .gradle, .idea, .vs, .vscode, .cursor, .config
- 保留 gradle
- 删除 temp_check（如果为空）
